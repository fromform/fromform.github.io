Installation​

#!/bin/sh

apt-get update

# install additional packages
apt install -y curl gnupg2 ca-certificates lsb-release \ debian-archive-keyring

# download and save nginx signing key
curl https://nginx.org/keys/nginx_signing.key | 
    gpg --dearmor \ | 
    tee /usr/share/keyrings/nginx-archive-keyring.gpg >/dev/null

# use lsb_release to automatically determine the OS and release name
OS=$(lsb_release -is | tr '[:upper:]' '[:lower:]')
RELEASE=$(lsb_release -cs)
echo "deb [signed-by=/usr/share/keyrings/nginx-archive-keyring.gpg] \ 
    http://nginx.org/packages/${OS} ${RELEASE} nginx" \ | 
    tee /etc/apt/sources.list.d/nginx.list

# install
apt-get update
apt-get install -y nginx
nginx # start nginx

Directories​

/etc/nginx/nginx.conf

  • Global settings for worker, tuning, logging, loading dynamic modules, references

/etc/nginx/conf.d/

  • Contains default HTTP server configurations.
    Files in this directory ending in .conf are included in the top-level http block from within the /etc/nginx/nginx.conf file. It’s best practice to utilize include statements.

/var/log/nginx/

  • default log location

Commands​

nginx -h - Shows the NGINX help menu.

nginx -v - Shows the NGINX version.

nginx -v - Shows the NGINX version, build information, and configuration arguments, which show the modules built into the NGINX binar

nginx -t - Tests the NGINX configuration.

nginx -T - Tests the NGINX configuration and prints the validated configuration to the screen. This command is useful when seeking support.

nginx -s signal - The -s flag sends a signal to the NGINX master process. You can send Signals such as stop, quit, reload, and reopen. The stop signal discontinues the NGINX process immediately. The quit signal stops the NGINX process after it finishes processing in-flight requests. The reload signal reloads the configuration. The reopen signal instructs NGINX to reopen logfiles.

nginx -s reload - Reloading the NGINX configuration without stopping the server.
