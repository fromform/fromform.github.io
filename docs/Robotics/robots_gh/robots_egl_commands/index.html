KRL Commands: Schunk EGL 90-PN

Three C# script components are available here, which can be used to generate KRL commands for the electrical gripper. Detailed documentation on how the gripper is configured on the KUKA controller can also be found here.

warning

During program execution, the gripper movement cannot be interrupted once the MoveTo command has been issued.

Grasshopper C# Scripts​

For compatibility purposes, legacy RH7 script components are also available. As of 14 January 2025, we recommend using the Robots Version 1.6.5 plugin, which can be installed via Rhino's Package Manager.

using Robots;
using Robots.Commands;

EGL Init:​

Initializes the gripper for use in the next program. This process resets all parameters for the gripper to 0, clears all errors, and attempts to execute its referencing routine.

This command should be connected to the Init Commands input of the Robots "Create Program" component.

danger

During the referencing routine, the gripper will fully close. This may cause damage to any mounted tools. Proceed with care.

// Initialize Gripper
private void RunScript(ref object Command_)
{
    var manufacturer = Manufacturers.KUKA;

    var commands = new List<Command>()
    {
        new Robots.Commands.Custom(
            "EGL_Init",
            manufacturer,
            "EGL_Init()", // KRL command
            null),
    };

    var cmdGroup = new Group(commands);
    Command_ = cmdGroup;
}

EGL Set:​

Provides functions to configure various parameters of the gripper. If no input is provided or the input value is set to 0, no command for that parameter will be generated.

This command should be connected to the Init Commands input of the Robots "Create Program" component. Additionally, you can modify any of the gripper's settings during a program run to adjust its behavior. To achieve this, supply the updated commands at the desired Robots.Target.

// Set gripper properties
private void RunScript(
	double _Speed,
	double _Acceleration,
	double _Jerk,
	double _Amperage,
	ref object Command_)
{        
    var manufacturer = Manufacturers.KUKA;
    var commands = new List<Command>();

    if(_Speed > 0)
    commands.Add(
        new Robots.Commands.Custom(
            $"EGL_Set_Speed_{(int)_Speed}",
            manufacturer,
            $"EGL_SetSpeed({_Speed})", // KRL command
            null));

    if(_Acceleration > 0)
    commands.Add(
        new Robots.Commands.Custom(
            $"EGL_Set_Acceleration_{(int)_Acceleration}",
            manufacturer,
            $"EGL_SetAcceleration({_Acceleration})", // KRL command
            null));

    if(_Jerk > 0)
    commands.Add(
        new Robots.Commands.Custom(
            $"EGL_Set_Jerk_{(int)_Jerk}",
            manufacturer,
            $"EGL_SetJerk({_Jerk})", // KRL command
            null));

    if(_Amperage > 0)
    commands.Add(
        new Robots.Commands.Custom(
            $"EGL_Set_Amperage_{(int)_Amperage}",
            manufacturer,
            $"EGL_SetAmperage({_Amperage})", // KRL command
            null));

    var cmdGroup = new Group(commands);
    Command_ = cmdGroup;
}

EGL MoveTo:​

Sets the desired position and movement command for the gripper.

This command must be supplied to each Robots.Target where you intend the gripper to open or close.

// Move to position
private void RunScript(double _Position, ref object Command_)
{        
    var manufacturer = Manufacturers.KUKA;

    Command_ = new Robots.Commands.Custom(
        $"EGL_MoveTo_{(int)_Position}",
        manufacturer,
        $"EGL_IS_OK = EGL_MoveTo({_Position})", // KRL command
        null);;
}

Result:​

In the left panel labeled '00.src', the commands provided by EGL_Init and EGL_Set are displayed. These commands are executed at the very beginning of the KRL program to initialize and prepare the gripper for use.

In the right panel labeled '01.src', it is shown that after each PTP movement, the gripper either fully opens or fully closes. By supplying values in the range of 0.0 to 85.0, intermediate positions can be set, enabling the gripper to accommodate workpieces of varying sizes.

EGL_IS_OK stores a boolean value indicating whether any exceptions occurred during the gripper's movement.
